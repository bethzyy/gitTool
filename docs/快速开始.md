# Git GUI 提交工具 - 快速开始

## 🚀 最简单的使用方法

### 方式 1: 双击启动（推荐）

直接双击以下任一文件：

1. **`启动应用.bat`** - 显示启动日志，然后打开应用窗口
2. **`启动应用(无窗口).bat`** - 直接打开应用窗口（推荐）
3. **`启动应用.vbs`** - 无需命令行窗口，最简洁

### 方式 2: 命令行启动

打开命令提示符（CMD），在当前目录运行：

```bash
python git_gui_app.py
```

## 📝 第一次使用

### 1. 准备工作

确保已安装：
- ✅ **Python 3.7+** - [下载](https://www.python.org/downloads/)
- ✅ **Git** - [下载](https://git-scm.com/downloads/)

### 2. 配置 Git（首次使用）

打开命令提示符，运行：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### 3. 在 GitHub 创建仓库

1. 登录 GitHub
2. 点击右上角 "+" → "New repository"
3. 填写仓库名称
4. 点击 "Create repository"

## 💡 使用步骤

1. **启动应用** - 双击 `启动应用.bat`

2. **填写信息**：
   - **Git 仓库地址**：例如 `https://github.com/用户名/仓库名.git`
   - **提交信息**：例如 "更新了用户登录功能"
   - **代码路径**：点击"浏览..."选择你的项目文件夹

3. **点击"提交到 GitHub"**

4. **等待完成**：
   - ✅ 自动扫描敏感信息
   - ✅ 自动执行 Git 操作
   - ✅ 自动推送到 GitHub

## ⚠️ 安全警告

如果应用检测到敏感信息（API Key、密码等）：
- 会弹出警告窗口
- 显示所有问题位置
- **阻止提交**，要求先处理安全问题

## 📂 生成的文件

运行后会创建：
- **`logs/`** 文件夹 - 存放运行日志
- **`logs/app-YYYY-MM-DD.log`** - 当天的日志文件

## ❓ 遇到问题？

### 应用无法启动

**检查 Python：**
```bash
python --version
```

如果没有显示版本号，需要安装 Python：https://www.python.org/downloads/

**检查 Git：**
```bash
git --version
```

如果没有显示版本号，需要安装 Git：https://git-scm.com/downloads/

### 推送失败

1. **认证错误** - 需要配置 GitHub 凭证
   - 推荐：使用 SSH 密钥
   - 备选：使用 Personal Access Token

2. **分支错误** - 应用已自动修复，会检测当前分支（main/master）

3. **无改动提交** - 如果代码没有修改，会提示"nothing to commit"

### 查看详细日志

打开 `logs` 文件夹，查看当天的 `.log` 文件，里面有详细的运行记录。
