# Git GUI 提交工具 - 桌面版

一个简洁的 Git GUI 桌面应用程序，帮助你轻松提交代码到 GitHub，并自动检测敏感信息。

## ✨ 特性

- ✅ **纯桌面应用** - 无需浏览器，直接运行
- ✅ **一键提交** - 自动完成 git add、commit、push
- ✅ **安全检查** - 自动扫描 API Key、密码、私钥等敏感信息
- ✅ **实时日志** - 所有操作都有详细日志记录
- ✅ **简洁界面** - 直观易用的图形界面

## 📋 前置要求

1. **Python 3.x** - [下载地址](https://www.python.org/downloads/)
   - 安装时勾选 "Add Python to PATH"
2. **Git** - [下载地址](https://git-scm.com/downloads)
3. **GitHub 账号** - 需要有一个 GitHub 仓库

## 🚀 快速开始

### 方法 1: 使用启动脚本（推荐）

双击运行 `run_app.bat` 即可启动应用程序

### 方法 2: 手动启动

1. 打开命令提示符（CMD）
2. 进入应用目录
3. 运行命令：
```bash
python git_gui_app.py
```

## 📖 使用方法

### 1. 填写信息

- **Git 仓库地址**: 输入你的 GitHub 仓库 URL
  - 示例：`https://github.com/username/repo.git`

- **提交信息**: 描述这次提交的更改
  - 示例：`更新了用户登录功能`

- **代码路径**: 选择要提交的代码文件夹
  - 点击"浏览..."按钮选择文件夹
  - 或直接输入路径，如：`C:\Users\YourName\project`

### 2. 点击"提交到 GitHub"

应用程序会自动：

1. ✅ **扫描敏感信息**
   - API Keys (OpenAI, AWS, etc.)
   - 密码和 Token
   - 私钥文件
   - 数据库连接字符串
   - Webhook URLs

2. ⚠️ **如果发现问题**
   - 弹出警告窗口
   - 显示所有敏感信息位置
   - **阻止提交**，要求先处理安全问题

3. ✅ **如果安全检查通过**
   - 初始化 Git 仓库（如果需要）
   - 添加所有文件到暂存区
   - 创建提交
   - 推送到 GitHub

4. 📝 **查看日志**
   - 所有操作都会显示在日志区域
   - 日志同时保存到 `logs/` 文件夹

## 🔒 安全检查详情

应用程序会自动检测以下敏感信息：

| 类别 | 检测内容 |
|------|----------|
| API Key | `api_key`, `sk_*` (OpenAI), `AKIA*` (AWS) 等 |
| 密码 | `password =`, `passwd =` 等 |
| Token | `bearer token`, `authorization` 等 |
| 私钥 | RSA 私钥、OpenSSH 私钥等 |
| 数据库 | MongoDB、MySQL、PostgreSQL 连接字符串 |
| Webhook | Slack、Discord Webhook URLs |

**自动忽略：**
- `node_modules`、`.git`、`venv` 等目录
- 示例代码（`YOUR_API_KEY`、`example` 等）
- 构建输出目录（`dist`、`build` 等）

## ❓ 常见问题

### Q: 提示"未找到 Python"
**A:** 需要先安装 Python 3.x
- 下载：https://www.python.org/downloads/
- 安装时务必勾选 "Add Python to PATH"

### Q: 提示"未找到 Git"
**A:** 需要先安装 Git
- 下载：https://git-scm.com/downloads
- 安装后可能需要重启电脑

### Q: 安全检查警告误报
**A:** 如果确信不是敏感信息，可以：
1. 将敏感代码移到单独的配置文件
2. 使用环境变量
3. 添加到 `.gitignore`

### Q: 推送失败，提示认证错误
**A:** 需要配置 GitHub 凭证：
- **推荐**: 使用 SSH 密钥
- **备选**: 使用 GitHub Personal Access Token

### Q: 提示"没有新的更改需要提交"
**A:** 说明所有文件都是最新的，无需提交

## 🔧 配置 Git 用户信息（首次使用）

如果首次使用 Git，需要配置用户信息：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

## 📝 提交信息规范建议

好的提交信息示例：
- ✅ `feat: 添加用户登录功能`
- ✅ `fix: 修复页面加载错误`
- ✅ `docs: 更新 README 文档`
- ✅ `style: 优化按钮样式`

不好的提交信息：
- ❌ `update`
- ❌ `fix bug`
- ❌ `test`

## 🎯 使用技巧

1. **首次推送前**，先配置好 Git 用户信息
2. **敏感信息**应该放在环境变量或配置文件中
3. **大型项目**建议先在 `.gitignore` 中排除不必要的文件
4. **查看日志**可以帮助排查问题

## 📞 获取帮助

如果遇到问题：
1. 查看 `logs/` 文件夹中的日志文件
2. 确认 Python 和 Git 已正确安装
3. 检查网络连接和 GitHub 仓库地址
